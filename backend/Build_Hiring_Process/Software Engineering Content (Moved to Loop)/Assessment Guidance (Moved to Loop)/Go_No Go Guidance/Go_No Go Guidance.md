# Go/No Go Guidance

## Interviewing at Level

We use the same leveling system provided by the [Career Development Framework (CDF)](https://twodegrees1.sharepoint.com/teams/Build/SitePages/Career-Framework.aspx) for interview candidates as we use for our employees. 

* The technical level mapping to CDF can be found here…
* The rubric for assessing a candidate’s level can be found here…

It is important that when assessing Go/No Go for a candidate, we are using the appropriate level for the assessment. For example, the expectations for a senior engineer in a given category is different than an engineer. If, when interviewing, you as the interviewer decide to change the level of the candidate for better fit, the assessment should be appropriately adjusted and the write up and feedback should clearly state the level change. 

## Pass/Fail

At each level, we evaluate pass/fail based on how a candidate performs given multiple technical evaluation categories. Pass/fail is assessed with the following criteria:

1. A score of good or better in at least all but one of the categories.
2. A score of good or better in required categories.

The rating scale for a candidate can be found here…

When providing a score that is below Good, it is important to fully assess the area. Do not make an assumption that a candidate does not meet requirements for a category by asking only questions in one area. For example, a candidate that does not express any knowledge of OOP, in itself does not mean the candidate fails Fundamentals. A broader set of questions is needed to accurately assess. 

You can find a broad list of sample questions to evaluate technical areas here.
