**Summary:** The Recruiter who set this interview up wasn't sure if the candidate was a Sr. Engineer or an Engineer. After talking with the candidate, I recommend that we move forward with the candidate as an Engineer. The candidate did well in most categories except for the Database portion, but I think the candidate will be able to provide velocity right out of the gate on a project. Here’s how the candidate did:

**Scale**: Not Covered / Poor / Fair / Good / Excellent

**Assessed as:**Engineer \[i.e. this is the role that your write up is for; i.e. this is where you state what role the scores below are assessed against]

**Communication: Good**

* Good communication skills and professional presence (on the phone)
* Good introduction and summary of background
* Handled ambiguous questions well; was transparent when candidate didn’t know an answer
* Asked clarifying question when didn’t understand; didn’t hedge / obfuscate

**Team / Culture Fit: Good**

* Career aspirations; wants to work as an engineer
* Flexibility; open to and interested in working on different stacks / tools (not tied to one technology)
* Goals: wants to grow into an Architect
* Why Slalom: Attracted to Slalom by the prospect of getting coaching and getting depth in different stacks

**CS / Core: Good**

* Got through OOP fundamentals (inheritance, polymorphism, encapsulation, Abstract Classes, Interfaces)
* Understood singleton pattern and was able to describe
* Understood and explained generics quite well (used a List\<t\> as an example)
* Understood basic threading and asynch calls (used an async database pull as an example)
* Data Modeling and Set Processing was good

**Database: Fair**

* Got basic SQL syntax and correctly wrote SELECTs
* Was able to use COUNT but struggled with GROUP BY
* Unfamiliar with NoSQL databases

**Web / Scripting / UI: Fair**

* Understands REST / JSON and SOAP / XML and pros/cons
* Did not know REST Verbs
* Was able to use JavaScript and RegEx to validate an email address
* Was able to articulate appropriate workloads for client and server

**Things to Check in person**

* System design
* Troubleshooting
* Web / JS coding
