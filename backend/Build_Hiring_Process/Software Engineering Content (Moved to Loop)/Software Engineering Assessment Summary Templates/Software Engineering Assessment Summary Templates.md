# Software Engineering Assessment Summary Templates

Please do not include any personally identifiable information (PII) about the candidate in your evaluation. 

If you have a question about what may be considered personally identifiable information, take the [**PII three\-minute mini\-lesson**](https://rise.articulate.com/share/ilwziedjqh98CncqQydOlfTqiaQHt63w#/)

We use the same Assessment Summary email template for both initial and technical interviews.

## Initial Interview

During the Initial Interview, we are more focused on trying to make an assessment of the level that the candidate is at, and making a decision on whether the candidate should be brought in for an in\-person interview.

**These are the categories that are currently being used to rate that ability:**

* Communication
* Team / Culture Fit
* Fundamentals


	+ CS / Core
	+ Algorithms
	+ Data Modeling
* Web / Scripting
* Troubleshooting
* Things to Check in next round (if there is one)

## Technical Interview (i.e. whiteboarding interview)

The Technical Interview on the other hand, is to assess the likelihood of the individual being able to deliver development velocity close to day 1 as well as their ability to solve novel problems, and to learn/grow in the role (aptitude).

Staffability and potential for development velocity on day 1 is indeed a litmus test that all candidates should pass. As an example, it may seem hard to pass up on smart person who is great with Mathematics. But, it is best that we do if they have not been doing what we do here and would need ramp up to deliver development velocity.

Every one of us will be learning everyday on the job, but for the most part, the expectation is that those investments can be made while providing useful velocity on projects.

**These are the categories that are currently being used to rate that ability:**

### Engineer / Senior Engineer

* Coding


	+ Code Writing
	+ Unit Testing
	+ Problem Solving (e.g. Performance and Storage constraints discussion)
* Object/Data Modelling


	+ Object Modelling
	+ Data Modelling
	+ SQL writing
* Basic Architecture (time\-permitting)


	+ High\-level App Architecture
	+ FE architecture
	+ BE architecture
* Communication / Culture
* Other


	+ Questions asked by the candidate (if any)
### Architect / Senior Architect

* Coding


	+ Code Writing
	+ Unit Testing
	+ Problem Solving (e.g. Performance and Storage constraints discussion)
* Architecture \& Design


	+ General Architecture / Design Patterns / Frameworks
	+ APIs/Services
	+ Data Modelling, Storage \& Access
	+ Cloud Principles \& Services
	+ Frontend / Mobile Clients
	+ Teaming / Delivery Discussion
* Technical Leadership Skills


	+ Communication
	+ Culture
	+ Leadership \& Mentoring
* Other


	+ Questions asked by the candidate (if any)
