# Performing coding assessments using Miro



| **Summary** | There are a couple of aspects of code interaction that can be evaluated using Miro: Can the candidate interpret or understand existing code? and can they reason through a problem and create some pseudocode. This page provides tips for utilizing Miro as a code evaluation tool |
| --- | --- |
| **Audience** | * Interviewers |
| **When** | On the Day of the interview |

# Overview

Miro is a very rudimentary coding evaluation tool. It is not a tool that will allow candidates or interviewers to compile, run or test code. It will be most useful for working with a candidate as they reason through either a SQL script or some pseudo code to create an algorithm or to analyze an existing code block that is provided by the interviewer.. Slalom is continuing to evaluate tools and processes that provide more robust capability in this area. This tip sheet will help maximize the effectiveness of Miro as a tool used to evaluate a candidate’s ability to work through the logic of a problem and develop an appropriate solution.

# Exercise 1: Can they interpret code

In this scenario, we are going to ask the candidate to look at a block of code and tell us what is going on with that block of code. This is a good exercise that should show how the candidate will do when working with pull requests/code reviews or in pairing with other team members.

## Setup

* Before the interview, Identify one or two code blocks that you will walk the candidate through. Consider code blocks with different types of issues or tricky logic.
* Library of code examples can be found here: [https://miro.com/app/board/o9J\_lDwySeY\=/](https://miro.com/app/board/o9J_lDwySeY=/)

## In Interview

* Ask the candidate if they understand what is happening in the code block
* Ask the candidate if they see opportunities for improvement with the code block: Are there inefficiencies or flaws in the code that should be addressed?

# Exercise 2: Can they write code

This is the classic ‘coding problem’ phase of an interview. In this scenario, we are going to ask the candidate to interpret a simple user story, algorithm diagram or statement and we will ask them to write some code to reflect that.

## Setup

* Identify one or two coding problems that you will use to assess the candidate’s ability. These should be coding problems that you understand and can assess the candidate’s experience with.
* These should be in line with the candidate’s role target.

## In Interview

* Create a text block for the candidate on the whiteboard space – should be wide enough with a mono\-spaced font selected. (see guidelines below)
* Ask the candidate to walk through their logical thought process as they create code to solve the problem – use arrows and highlighter to ask about sections of code.
