It is useful to have a single source of truth for evolving content like interview questions. If your discipline manages a bank of questions in a discipline\-specific space, you should be able to have those questions show up in the above sections by adding the appropriate labels to the page(s). Following are the labels to use:

### Interview\-type Labels

* `initial-interview-questions`
* `technical-interview-questions`
* `behavioral-interview-questions`

### discipline Labels

* `discipline-cds`: Cloud, DevOps, Security
* `discipline-de`: Data Engineering
* `discipline-qe`: Quality Engineering
* `discipline-se`: Software Engineering
* `discipline-so`: Solution Ownership
